# 自然写作指南：去AI味 + 加人味

写自媒体文案时，既要去掉机器味，又要加上人味。遵循以下原则。

---

## 第一部分：去AI味（减法）

### 1. 打破工整结构

**避免**：
- 机械分点（1,2,3...）
- 每段固定格式（首先...其次...最后...）
- 过度使用小标题

**改用**：
- 自然段落过渡
- 偶尔用不完整的句式
- 让结构服从内容

```
❌: "首先，我们需要理解X的概念。其次，掌握Y的方法。最后，应用到实际场景中。"
✅: "要理解X，得先搞清楚它跟Y的关系。弄明白这个，后面的实际应用就顺了。"
```

### 2. 删除废话

**常见废话**：
- "在当今这个...的时代"
- "综上所述"、"总而言之"
- "显著"、"有效"、"重要"（不说具体程度）
- "值得注意的是"、"令人欣喜的是"

**改为**：
- 直接说结论
- 用具体数据代替模糊词
- 删除所有不影响理解的词

```
❌: "这个方法非常有效地提升了系统性能。"
✅: "这方法让系统快了30%。"
```

### 3. 情感要真实

**避免**：
- 假热情（"这真是太棒了!"）
- 永远积极向上
- 刻意的感叹号

**改用**：
- 客观陈述事实
- 允许中性或负面评价
- 真实的困惑和疑问

```
❌: "令人激动的是，这个工具为我们带来了前所未有的便利!"
✅: "这工具确实省事，虽然有些功能还不太稳定。"
```

### 4. 不堆砌辞藻

**避免**：
- "受益匪浅"、"醍醐灌顶"、"如沐春风"
- "深刻独到"、"鞭辟入里"
- "在当今这个快速发展的时代"
- "综上所述"、"值得一提的是"

**改为**：
- 直接说事
- 能不说就不说
- 用大白话表达

```
❌: "读完这本书，我醍醐灌顶，受益匪浅。"
✅: "这本书改变了我看问题的方式。"
```

## 第二部分：加人味（加法）

### 1. 具象 > 抽象

**原则**：用场景、细节、动作代替概括性描述

**避免**：
- 抽象的感受词（"受益匪浅"、"深受启发"）
- 宏观的陈述（"这本书非常有价值"）

**改用**：
- 具体场景（"读到第三章时，我停下了"）
- 身体反应（"这句话击中了我"、"我攥紧了拳头"）
- 具体细节（"下午3点的阳光照在书页上"）

```
❌: "阅读这本书让我受益匪浅，作者的见解深刻独到。"
✅: "读到第三章时，作者说'恐惧是成长的必要条件'，我突然停下来。这句话让我想起了去年做项目时的某个下午..."
```

### 2. 个人化 > 通用化

**原则**：用"我"的经历和视角，说"我"的故事

**避免**：
- "人们都说"、"很多人都觉得"
- "我们"、"大家"这种泛化代词
- 可以套用到任何人身上的话

**改用**：
- "我是"、"我经历过"
- 说具体的个人故事
- 说真实的个人困惑

```
❌: "作为程序员，我们都面临职业焦虑。"
✅: "我做了10年FPGA工程师，去年看到AI能写RTL代码时，我手抖了一下。"
```

### 3. 动词 > 形容词

**原则**：用动作表达情绪，而不是用词描述情绪

**避免**：
- "我很愤怒"、"我很感动"
- "美丽的"、"震撼的"、"深刻的"
- "令人惊叹"、"令人欣喜"

**改用**：
- "我攥紧了拳头"（愤怒）
- "我读了两遍"（感动）
- "窗外的风停了"（震撼）

```
❌: "这个故事让我非常感动。"
✅: "读到结局那页，我坐了10分钟没动。"
```

### 4. 短句 + 留白 > 长句 + 铺陈

**原则**：有呼吸感，给读者思考空间

**避免**：
- 一句话包含3个以上分句
- 排比句式
- 过度的修饰语

**改用**：
- 短句，5-15字一句
- 用"..."表示停顿
- 省略号代替完整表达
- 不把话说满

```
❌: "这个工具不仅功能强大，而且易于使用，更重要的是它的界面设计非常人性化，大大提升了用户体验。"
✅: "这工具挺好用。界面简单，不用看文档就能上手。"
```

### 5. 对比 > 独白

**原则**：用"过去vs现在"制造张力

**结构**：
- 过去：我之前是怎么想/做的
- 触发：发生了什么事/看到了什么
- 现在：我现在的想法/做法变了

**示例**：
```
过去：我总觉得要准备好了再开始。做项目前文档、规范、脚本都准备齐全，才敢动工。

变化：读了《原子习惯》，作者说"每天进步1%，一年后强37倍"。

现在：现在我先做起来，边做边调整。
```

## 第三部分：语言风格

### 用词原则

- **动词优先**："我停下了" > "我感到很震撼"
- **名词具体**："键盘" > "设备"、"工具"
- **少用程度词**："很"、"非常"、"特别"（能删就删）
- **少用连接词**："因为...所以..."（让关系自然浮现）

### 句式节奏

- 3-5个字一句：有冲击力
- 10-15字一句：正常叙述
- 偶尔极短：2-3字，制造节奏

**示例**：
```
我做了10年FPGA。
去年看到AI能写代码。
我手抖了一下。
```

### 允许的表达

- 不完整句："嗯，就是..."
- 口语词："其实"、"反正"、"差不多"
- 括号补充：（就像现在这样）
- 省略号："..."表示停顿、未尽之意

## 第四部分：常见场景模板

### 读书推荐类

**结构**：
1. 一个场景/经历（为什么读这本书）
2. 一个触发点（书中某句话/某个观点击中你）
3. 一个改变（你现在的想法/做法变了）
4. 一个具体建议（谁应该读这本书）

**示例**：
```
我做FPGA工程师10年，总觉得要准备好了再开始。做项目前文档、规范、脚本都准备齐全，才敢动工。

直到读《原子习惯》，作者说"每天进步1%"，我才明白自己是在拖延。

现在：现在我先做起来，边做边调整。

如果你也是总想"准备好了再做"，这本书会帮你。
```

### 个人故事类

**要素**：
- **具体时间**："上周五"、"去年冬天"
- **具体地点**："公司楼下"、"咖啡馆角落"
- **具体动作**："我盯着屏幕"、"我放下咖啡杯"
- **具体细节**："下午3点的阳光"、"键盘的声音"

### 观点输出类

**方法**：
- 先说结论
- 用1个个人经历支持
- 说"我觉得"、"我认为"，不装权威
- 允许不确定："这块我也没完全搞清楚"

## 第五部分：和AI文字的区别

| 维度 | AI文字 | 有人味的文字 |
|------|--------|-------------|
| 句长 | 长句，结构完整 | 短句，有留白 |
| 词汇 | 华丽、准确 | 简单、真实 |
| 情绪 | 假积极、永远正能量 | 允许困惑、犹豫、不确定 |
| 经历 | 通用、可以套用 | 具体、只有你有 |
| 形容词 | 很多 | 很少 |
| 动词 | 一般 | 很多 |
| 结构 | 工整、对称 | 自然、有节奏 |

## 第六部分：实际操作步骤

写文案时，按这个流程：

1. **先写草稿**：把想说的都写出来，不管多烂
2. **圈出抽象词**：受益匪浅、深刻独到、震撼、感动...
3. **替换成场景**："我感到震撼" → "我停下了，坐了10分钟没动"
4. **圈出形容词**：美丽的、强大的、高效的...
5. **删减连接词**："因为...所以..." → 直接说事
6. **拆长句**：一句话里超过3个逗号就拆
7. **删废话**："在当今"、"综上所述"、"值得注意的是"
8. **删AI味**：首先...其次...最后...、值得注意的是、令人欣喜的是
9. **检查节奏**：读一遍，不顺就改

## 最终检查清单

写完文案后，检查：

- [ ] 至少有1个具体场景/经历
- [ ] 没有"受益匪浅"、"深刻独到"等空话
- [ ] 没有"首先...其次...最后"的固定模式
- [ ] 删除了所有"在当今"、"综上所述"类的废话
- [ ] 形容词少于5个
- [ ] 动词多于形容词
- [ ] 有"我"的视角和经历
- [ ] 句长有变化，不是每句都一样长
- [ ] 有短句（5字以内）
- [ ] 有省略号或括号
- [ ] 读起来像人在说话
- [ ] 没有刻意的感叹号和假热情

---

## 核心记住一点

**不要试图写出"完美的文章"，要写出"真实的文章"。**

好文笔的标准不是"辞藻华丽"，而是"有人看进去，有人记得住"。

AI能写出标准化的好文章，但只有你能写出只有你能写出的东西。
